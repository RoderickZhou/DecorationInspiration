# 装修灵感日报工具 PRD v1

## 1. 产品定位

这是一个面向家庭装修决策早期阶段的灵感推荐工具。

目标用户当前对装修认知有限，不急于马上定风格或施工方案，而是希望系统每天自动采集高质量装修内容，经过筛选、摘要和个性化整理后，生成一份可以在 2-3 分钟内浏览完的“装修灵感日报”。

第一版产品不追求完整装修平台能力，优先解决以下问题：

- 自动收集装修灵感内容
- 自动筛选出更适合用户家庭画像的案例
- 自动生成日报网页
- 支持用户收藏、喜欢、不喜欢
- 将用户反馈沉淀为后续推荐依据

## 2. 目标用户

### 2.1 当前用户画像

- 房屋建筑面积约 132 平，到手面积约 98 平
- 四口常住
- 大儿子 7 岁，小儿子 8 个月
- 纯自住，长期居住，不考虑置换
- 倾向 3 室布局
- 高频做饭，厨房与餐边柜优先级高
- 全屋收纳都重要
- 希望在客厅保留自己的功能角
- 当前对装修了解较少，需要通过大量案例图片建立判断力
- 计划 2028 年初开始装修

### 2.2 用户阶段特征

- 当前阶段更需要“看懂案例”，不是“立刻做设计图”
- 愿意每天花 2-3 分钟浏览精选内容
- 能接受通过收藏和跳过的方式训练系统偏好

## 3. 核心目标

### 3.1 业务目标

- 建立一个可持续增长的装修灵感知识库
- 通过每日推荐培养用户装修判断力
- 为后续风格推荐、空间规划、采购建议打基础

### 3.2 用户目标

- 每天看到少量但高质量的装修内容
- 每条内容信息负担轻，主要看图，配简短解释
- 系统逐渐理解“什么适合我家”
- 逐步解决儿童房、客厅功能角、厨房收纳等关键问题

## 4. 第一版范围

### 4.1 第一版要做

- 多来源自动采集
- 内容去重与初筛
- Minimax 自动摘要、标签、适配度判断
- 日报 HTML 页面生成
- 收藏 / 喜欢 / 不喜欢 / 跳过反馈
- 本地 JSON 数据交换

### 4.2 第一版先不做

- 完整装修公司评分系统
- 完整比价系统
- 施工图、预算表、合同分析
- 复杂账号体系
- 云端多端同步
- 实时聊天式 AI 装修顾问

## 5. 使用流程

### 5.1 系统流程

```text
定时任务触发
-> 多来源采集新内容
-> 原始内容入库
-> 去重与规则初筛
-> Minimax 做摘要/标签/适配判断
-> 生成当日推荐集合
-> 输出 report.json
-> 前端读取 report.json 生成日报页面
-> 用户浏览并点击喜欢/不喜欢/收藏/跳过
-> 反馈写入 actions.jsonl
-> 下次生成推荐时使用反馈优化结果
```

### 5.2 用户流程

```text
打开日报页面
-> 浏览 12-20 条精选卡片
-> 对感兴趣内容收藏或标记喜欢
-> 对明显不适合内容标记不喜欢
-> 次日查看新的日报结果
```

## 6. 采集来源策略

### 6.1 第一优先级来源

- 小红书
- 好好住
- 一兜糖
- Pinterest
- 花瓣

适合产出用户每天浏览的主内容流，重点是实景、普通家庭、空间落地感。

### 6.2 第二优先级来源

- Houzz
- Dezeen
- ArchDaily
- gooood

适合补充趋势、审美拓展和更强设计感的案例。

### 6.3 第三优先级来源

- Behance
- Dribbble
- 拓者设计吧
- 如室
- AD

适合补充高端案例和概念灵感，不作为日报主粮来源。

## 7. 推荐筛选原则

### 7.1 高优先级主题

- 二孩家庭
- 儿童房
- 中小三居
- 客厅功能角
- 高频做饭厨房
- 餐边柜
- 收纳强化
- 长期自住

### 7.2 推荐排序因子

- 与用户家庭画像匹配度
- 图片质量与可读性
- 内容是否为真实落地案例
- 互动数据质量
- 文本可摘要程度
- 重复度和模板化程度

### 7.3 降权内容

- 纯营销内容
- 图片少且信息量低
- 明显不适合有孩子家庭
- 过度概念化、难落地
- 与用户面积和居住方式偏差过大

## 8. Minimax 角色定义

Minimax 2.7 cn 在第一版中负责“内容审核与整理”，不直接负责采集。

### 8.1 Minimax 负责内容

- 判断是否值得入选日报
- 为内容生成一句短摘要
- 提取风格、空间、家庭、特征标签
- 给出适配度分数
- 解释适合或不适合用户家庭的原因
- 生成日报底部总结

### 8.2 示例输出要求

输入是一条采集内容，Minimax 需要输出：

- `summary`
- `fit_score`
- `fit_reason`
- `risk_notes`
- `tags.style`
- `tags.space`
- `tags.family`
- `tags.features`

## 10. 页面设计

### 10.1 页面目标

让用户在 2-3 分钟内完成今日浏览，不产生信息疲劳。

### 10.2 页面结构

- 顶部摘要区
  - 今日新增素材数
  - 筛选后推荐数
  - 今日重点主题
  - 一句日报摘要
- 推荐卡片区
  - 每页 12-20 张卡片
  - 卡片展示图片、标题、短摘要、标签、交互按钮
- 底部总结区
  - 今日趋势
  - 今日适合用户家的重点
  - 今日问题

### 10.3 卡片交互

- 喜欢
- 不喜欢
- 收藏
- 查看原文

按钮点击后立即本地记录反馈，不阻塞浏览。

## 12. 推荐逻辑 v1

### 12.1 输入

- 用户固定画像
- 最近一段时间收藏/喜欢/不喜欢反馈
- 当日采集候选内容

### 12.2 输出

- 当日推荐卡片列表
- 今日重点主题
- 一句话日报摘要
- 今日问题

### 12.3 简化版排序公式

```text
最终分 = 画像匹配度 * 0.45
      + 互动质量 * 0.20
      + 图片质量 * 0.10
      + 内容清晰度 * 0.10
      + 用户反馈相似度 * 0.15
```

## 13. 技术方案建议

## 14. 成功标准

### 14.1 用户体验标准

- 用户每天愿意打开一次
- 用户能在 2-3 分钟内浏览完成
- 用户每周能积累稳定的收藏样本

### 14.2 系统效果标准

- 每日推荐内容中至少 70% 被用户认为“值得看”
- 不喜欢内容比例逐步下降
- 与儿童房、客厅功能角、厨房收纳相关的内容质量持续上升

## 16. 当前结论

第一版最重要的不是“覆盖所有平台”，而是把下面这条链路跑顺：

`自动采集 -> 自动筛选 -> 自动摘要 -> 生成日报 -> 用户反馈 -> 次日更准`

只要这条链路稳定，后面无论接更多平台、接更强模型，还是扩展到装修决策系统，都有坚实基础。
