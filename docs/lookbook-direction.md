# 产品方向调整：从「装修灵感日报」到「视觉灵感库 Lookbook」

**调整日期**：2026-05-30
**触发**：用户打开真实数据 Minimax 跑出的日报页后反馈「感受不到这个项目能给我提供什么帮助」。

## 调整前

「装修灵感日报」：
- 每天 5 条 AI 摘要的卡片
- 大量 AI 文本（今日焦点 / question_of_the_day / fit_direction）
- 假设用户每天 3 分钟主动来读
- 假设用户重视 AI 解读多于图片

## 用户实际状态

经过三轮提问明确：

| 维度 | 状态 |
|---|---|
| 现在怎么"看装修" | 1) 刷小红书/抖音 → 截图  2) 主动用 Pinterest/花瓣/好好住 收藏  3) 跟太太讨论时才看 |
| 痛点主因 | AI 写文字虽专业但用不上 / 看不出比直接刷小红书强在哪 / 看完不知道该干啥 |
| 期望核心能力 | **视觉灵感库：按空间分类的海量真图，能刷能存** |

明确拒绝：决策助手、审美图谱、让 AI 推荐。

## 调整后

「装修视觉灵感库 Lookbook」：
- 瀑布流大图为主，文字辅
- 按空间（儿童房 / 客厅 / 厨房 / 收纳 / 主卧）分 tab
- 按风格（原木 / 简约 / 奶油 / 侘寂…）chip 筛
- 默认按"适配你家的分数"排序，但适配分只是排序辅助，不强推
- 鼠标点 ♡ 加入跨平台收藏（localStorage），右上角收藏夹
- 点图直接跳原页

## 与小红书的差异化

| 维度 | 直接刷小红书 | Lookbook |
|---|---|---|
| 来源 | 单一平台 | 多平台聚合（v0.1 是 360 图搜聚合全网，v0.2 起接好好住/Pinterest，v0.3 起接小红书） |
| 组织 | 按 hashtag 散列 | 按你家关心的 5 个空间硬归类 |
| 个性化 | 算法黑盒 | 用你家画像（132平/二孩/高频做饭）显式打适配标记 |
| 收藏 | 平台内孤岛 | 跨平台统一收藏夹，长期累积，可导出 |

## v0.1 实现要点

1. **数据源**：360 图片搜索 JSON API（image.so.com/j）。
   - 优势：国内网络直达、公开接口无需登录、缩略图走 HTTPS CDN（qhimgs1）稳定可加载
   - 局限：标题偏 SEO 风、原页 URL 质量参差、图源混杂（含部分 AI 生成图，已用 `--skip-aigc` 过滤）
   - 后续会接的中文真案例平台（好好住 / 小红书）反爬较重，需要 DrissionPage 实浏览器栈，是独立的硬骨头，分开做
2. **AI 处理**：复用现有 `run_item_structuring.py --mode minimax`（M2.7）。
   - 输出 tags + fit_score + risk_notes
   - long-form 字段（summary / fit_reason / why_selected）仍生成但前端不主推
3. **前端**：新建 `demo/lookbook.html`（vanilla JS + CSS columns 瀑布流，无第三方依赖）。
   - 原 `demo/renovation-daily-demo.html` 保留备查
4. **数据格式**：仍用现有 `report.json` schema（不破坏契约），UI 只消费 `items[*]` 字段

## 不做的事（明确边界）

- 暂不接小红书真实采集（反爬强，需 DrissionPage + cookies，独立 sprint）
- 暂不做"决策助手 / 审美图谱"等 AI 重交互形态（用户已拒绝）
- 暂不动 `web/` React 工程（先用静态 HTML 验证形态）
- 暂不做反馈学习闭环（先看「刷」做爽不爽）

## 验收方式

让用户用 2 分钟刷一下 lookbook，对比直接刷小红书：
- 强：单页就够刷、按空间归类、跨平台聚合潜力、收藏不丢
- 弱：v0.1 阶段图源还不是"小红书级别"的真案例，分辨率参差，标题 SEO 味重
